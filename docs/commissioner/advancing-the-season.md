# Advancing the Season

The commissioner controls when the league moves forward. Coaches submit their plans; you review, lock, and advance.

## The weekly loop

```
Unlock submissions
      ↓
Coaches submit game plans + recruiting actions
      ↓
Monitor Submissions section until all coaches are Ready
      ↓
Lock submissions
      ↓
Advance the week in the Mac app
      ↓
Repeat
```

## Monitoring submissions

The **Submissions** section in League Hub shows every coach who has a team assigned:

| Indicator | Meaning |
|-----------|---------|
| Green checkmark | Game plan submitted |
| Red X | Game plan not yet submitted |
| Recruiting count | Number of recruiting actions logged this week |
| **Ready** badge | Game plan submitted and at least one recruiting action logged |

You can advance once everyone shows **Ready**, or after the deadline passes. The decision is yours — you can advance with partial submissions if needed.

## Locking submissions

Before advancing, lock submissions so coaches can't change their plans mid-simulation:

**League Hub → Season Control → Lock Submissions (toggle on)**

Locked submissions appear as read-only in the coach portal. Coaches can still view their submission but cannot edit it.

To reopen submissions (e.g. if a coach needs to fix a mistake): toggle **Lock Submissions** off.

## Advancing the week

Advancing is done in the **Mac app**, not the server. The server stores what coaches submitted; the Mac app runs the simulation.

1. Confirm all submissions are locked
2. In the Mac app, go to **Season Hub**
3. Use the advance controls to run the week — this processes game results, applies recruiting actions, and moves the calendar forward
4. The server receives the updated state automatically
5. Unlock submissions to open the next week

## Season phases

The season moves through several phases:

| Phase | What's happening |
|-------|-----------------|
| **Preseason** | Roster finalization, depth chart setup |
| **Regular season** | Weekly game cycles, recruiting actions |
| **Bowl/Playoffs** | Postseason game weeks |
| **Offseason** | Signing Day, transfer portal, coaching carousel, draft |

The Mac app controls phase advancement. Each phase transition is a deliberate commissioner action.

## Offseason and token policy

When the season ends and the league enters offseason, the **Token Policy** setting determines whether coaches need to rejoin next season:

- **Carry Over** — coaches keep their tokens; they reconnect automatically next season
- **Reset** — tokens are invalidated at offseason; coaches must rejoin with a new join code

The current token policy is shown in League Hub during offseason. Change it in **Settings → Multiplayer** before advancing to offseason.
